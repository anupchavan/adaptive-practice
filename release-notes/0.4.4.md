## Added

- **Cross-topic synthesis, grounded.** When a session holds two or more notes, the plugin computes their *structural* connections from the existing index — notes that link to each other, shared tags, shared concept candidates — and hands the model a tiny bridge list with permission to write 1–2 synthesis questions that need both topics, only where a bridge is real. No bridge, no block, no forced analogies, ~zero extra tokens.

## Changed

- **Stems got their brevity back.** 0.4.3's self-containment rule caused paragraph-length questions. The rule now demands self-contained *and* terse: define what's used, 1–3 short sentences of context, cut every word that doesn't change the answer, and one question = one ask (no more "which statement identifies X *and also explains* Y").
- **The source note is revealed after answering, not before.** The "From [[note]]" chip moved into the result panel: naming the topic up front hands you the concept's category, and figuring out which concept applies is part of the exercise. The prompt no longer asks for inline links at all, and instructs the model to name the tested concept only when the question is unanswerable without it.

## Fixed

- **Links to notes that don't exist.** Models invented wikilinks to concepts ("[[Circuit Switching]]") that aren't notes, rendering as "click to create" bait. Unresolved internal links in questions, options, and explanations are now unwrapped to plain text at render time, and the prompt forbids model-written links entirely.
- **Longest-option-is-correct giveaway.** New hard rule: options must be homogeneous in form, reasoning depth, and length (within ~25%), written by drafting the correct option first and matching each distractor to it. The built-in format exemplar itself modeled this bias (its correct option was the longest) — fixed, its correct option is now the shortest. The eval metrics also track the giveaway per question (`correctLongestOption`) so provider comparisons can measure it.
