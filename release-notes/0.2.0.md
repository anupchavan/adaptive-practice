### Bug Fixes
- Removed the accidental `obsidian-custom-views` submodule entry from the repository so BRAT installs only Adaptive Practice files.
- Switched provider key entry to Obsidian secret storage UI instead of a custom password field, while preserving per-provider secret names.
- Added an explicit confirmation modal before closing an unfinished generated quiz so accidental outside clicks do not discard a session silently.
- Renamed ambiguous dashboard wording from "Ready for review" to scheduler-accurate due-note language.
- Improved provider error messages for invalid models, JSON-mode mismatches, missing keys, local endpoints, and provider-side failures.

### Improvements
- Added multi-provider BYOK generation support for Gemini, Anthropic, OpenAI, DeepSeek, Qwen, OpenRouter, and custom OpenAI-compatible endpoints.
- Added structured note understanding for generation: frontmatter, tags, links, headings, section excerpts, images, SVG notes, remote media, and provider-compatible PDFs.
- Added independent difficulty and concept calibration so model labels are reviewed by the plugin, one-step questions are downgraded, and title-framed recall can be rejected.
- Added flow-oriented practice behavior: easy batches request harder top-up questions, fluent correct answers pull harder questions forward, and misses recover to easier questions.
- Added daily practice scheduling with streak credit, meaningful-session checks, due-note selection, spaced review, fluency tracking, and a persistent dashboard.

### Tests
- Added 61 core tests covering provider parsing/errors, secret-name migration, difficulty calibration, concept anchoring, adaptive flow ordering, daily scheduling, media handling, markdown rendering, and release metadata.
- Added release validation to ensure `manifest.json`, `package.json`, `versions.json`, and build artifacts stay consistent before publishing.
