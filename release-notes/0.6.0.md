Two more analytical moves mined from a top-tier physics problem book, and a full pass over the plugin's chrome: settings, modals, and notices now speak fluent Obsidian.

## Added

- **Two new analytical moves (catalog: 24)**, mined from Pathfinder-style olympiad exercises and domain-general as always: **instrument vs reality** — a quantity known only through a recording process that systematically distorts it (delay, relative reference, sampling bias); recover the truth from the recording or predict the recording, against distractors that quietly equate the two — and **tight bounds** — inputs known only within ranges; derive the tightest range or extreme value the data licenses, against falsely precise points and looser-than-needed bounds.
- **Model picker.** The model name is now a dropdown of known-good models per provider (provider default marked), with a Custom option that reveals a free-text field — no more typing exact model IDs from memory.
- **Declarative settings on Obsidian 1.13+.** The settings tab now provides setting definitions to newer Obsidian versions, which makes every option findable from the global settings search; older versions keep the classic tab unchanged (minimum app version stays 1.11.4).

## Changed

- **Note filters upgraded** to the newer engine: new operators (is exactly, does not start/end with), new fields (basename, extension, outgoing links), inline suggestions for files, folders, tags, and property values, wikilink-aware value pills, and keyboard-friendly editing. Existing saved filters keep working unchanged.
- **Notices got a proper visual language**: a small plugin label, the message, and a right-aligned action bar with accent-styled primary buttons — replacing the crammed grey buttons. Notices also no longer swallow clicks around them.
- **Modals now use Obsidian's official building blocks** (modal titles, section headings, button rows) instead of hand-rolled headings and buttons, so they follow your theme correctly.
- Settings copy rewritten to Obsidian's conventions: shorter names, sentence case, concise descriptions.

## Fixed

- **Answering the last buffered question mid-generation no longer ends the session.** The action button now shows a "Generating next question…" loading state with a spinner and re-enables the moment the next flow batch lands.
- Verified the daily reminder end to end (minute-level check, time gate, once-a-day and cooldown suppression, draft/generation guards) — it works; its notice simply looked bad, and now doesn't.
