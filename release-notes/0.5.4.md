## Fixed

- **High-skill sessions no longer die with "Failed to generate enough challenging questions."** Any note above skill 80 put the whole session behind a strict difficulty gate: zero easy questions allowed per high-skill topic, a minimum hard count, subtopic spread — and if two repair attempts couldn't satisfy all of it, the plugin threw the entire session away. One question that the difficulty estimator relabeled "easy" was enough. Difficulty targets now shape effort instead of gating service: the challenge-repair requests still fire (so the model is still pushed to produce harder questions), but whatever the strongest available batch is, it ships — with the shortfall logged to the console. A slightly-softer session beats an error toast every time, and the in-session flow controller adapts difficulty from your results regardless.
