## Fixed

- **Claude output-token truncation, actually fixed.** 0.4.1 chunked large sessions, but the real culprit was elsewhere: newer Claude models (Sonnet 5 and later) run *adaptive thinking by default* when no `thinking` parameter is sent, and thinking tokens count against `max_tokens` — so the model silently spent the output budget thinking and truncated the question JSON, even on small batches. Question generation now explicitly disables thinking (the schema already forces reasoning before each answer via the explanation field, so nothing is lost), with a self-healing retry for models that reject the setting.
- Always-on-thinking models (Claude Fable/Mythos family) can't disable thinking, so they get extra `max_tokens` headroom (16K) instead.
- All Anthropic generation requests now floor `max_tokens` at 8K regardless of batch size — it's a cap, not a spend, so unusually long questions no longer truncate.
