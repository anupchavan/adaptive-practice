## Added

- **Answer verification (on by default).** Every generated batch is re-solved *blind* — the model gets the questions back with answers hidden and must derive each one independently. Questions whose marked answer fails the check are dropped, so a wrong answer key can no longer mark your correct choice incorrect. Costs about one extra request per batch; toggle under Settings → Verify answers. If the verifier contests more than half a batch, the verifier run is distrusted and the batch is kept.
- **Source-note link on every question.** A "From [[note]]" chip renders under the question header (clickable, with hover preview), so the source is one click away even when the model forgot to link it inline. The prompt now also asks for an inline wikilink at the first natural mention.

## Changed

- **Questions must be self-contained.** The prompt now requires every stem to define its variables and restate the problem setup it reasons about — you should be able to answer from concepts plus the stem, never from remembering the note. Numeric instances must be non-trivial: values a learner can answer from familiarity (4th root of 81) are ruled out unless triviality is the trap.
- State-tracing questions must cross-check their answer against every fact the stem asserts before shipping.

## Fixed

- Bare `==` comparisons outside code (e.g. `arr[low]==arr[mid]==arr[high]`) rendered as yellow highlight in Obsidian (`==text==` is highlight syntax). Prose tokens containing `==` are now backtick-wrapped mechanically, and the prompt warns the model about the collision.
- A time-dependent test fixture (practice-draft expiry) that started failing once its hardcoded date aged past the 7-day draft window.
