## Adaptive Practice 0.4.0

The research-grounding release: adaptive flow sessions, real spaced-repetition scheduling, and question quality that works for any subject — not just the vault it was developed in.

### Flow sessions (new)

- Sessions now start after a small opening batch and generate the rest **in the background while you answer**, adapting to how the session is going. A rolling controller keeps your success rate near the ~80–85% band that learning research says maximizes both retention and engagement — stepping difficulty up after fast, accurate streaks and easing off after misses, never whiplashing.
- Every follow-up batch knows what was already asked, so nothing repeats within a session. If generation fails mid-session, the session simply ends at the questions you have. Turn "Adaptive flow generation" off in settings to generate everything up front.

### Question types

- **Select-all-that-apply questions** (new): 4–5 options with two or more correct answers, graded as an exact set (order never matters), with a "Select all that apply" label, per-option correct/incorrect highlighting, and A–E keyboard toggling.
- Integer and decimal answers continue to grade with sensible tolerance, and the model is now told to prefer them whenever the answer is a number you should compute rather than recognize.

### Scheduling on real memory science

- Review intervals follow an FSRS-style forgetting curve, and every **subtopic now carries its own memory strength** — a note whose weakest concept is fading comes back before the note as a whole is due, with the reason shown.
- Editing a practiced note partially resets its stability (new content = partially invalidated memory) and brings it back soon.
- Sessions interleave questions across notes, at most three never-practiced notes join a daily session once you have review history, and a **"Review intensity"** setting (80–95% target recall) controls how aggressively things come back.
- Your streak now shows in the daily reminder and in a clickable status-bar item (`Practice 🔥 3d · 2 due`).

### Question quality, for any subject

- The generation prompt no longer carries any subject-specific assumptions — depth is derived from what mastery means in *your* note's field (procedural, quantitative, conceptual, or factual material), so history notes get meaningful fact questions while technical notes get construct/predict/debug questions.
- A **"Practice intent"** setting (durable mastery / exam cram / broad review) shapes question style to what you're studying for.
- Deterministic guards reject answer-leaking stems, near-duplicate options, and re-skinned batches that repeat one scenario; distractors must each embody a specific, nameable mistake.

### Consistency and cost

- Strict structured output on every provider that supports it (OpenAI, OpenRouter, Gemini), with automatic graceful fallback when a gateway rejects the schema.
- A format exemplar plus math-brace repair keep LaTeX, code blocks, and options rendering uniformly across models.
- Output tokens now scale with the question count instead of always paying the maximum, and Anthropic requests cache the shared system prompt.

### Fixes and polish

- Keyboard shortcuts only act when the practice pane is active — typing in another note can no longer select answers.
- Vault edits refresh the practice index within about 30 seconds instead of waiting for the half-hour sweep.
- Removed an orphaned legacy quiz UI; new "Export practice dataset" command writes your accumulated learning signals as JSONL.
